# Research: Elevation Graph Bug - Missing Start/Mid Sections

**Date**: 2026-01-24T22:57:36+01:00

## Research Question

There is an error in the elevation graph which makes the graph skip the start or mid section of a route. Visually, the graph does not start at 0 on the X axis, or there is a large distance between two points in the middle of the graph. Research the code responsible, the test suite setup, and develop a strategy for discovering the bug with tests (preferring fast unit tests).

## Summary

The elevation graph is generated by three key functions in `components/map.tsx`:
1. `generateElevationData()` - Main sampling function
2. `interpolatePointAtDistance()` - Point interpolation along route
3. `calculateDistance()` - Haversine distance calculation

**Most likely bug cause**: The `generateElevationData()` function **skips points** when `queryTerrainElevation()` returns null/undefined (lines 181-187). This creates gaps in the elevation data, causing the X-axis to not start at 0 or have large jumps in the middle.

**Secondary concern**: Only `directions[0]` is processed, potentially ignoring multi-segment routes.

## Detailed Findings

### 1. Elevation Data Generation (`components/map.tsx:150-195`)

The `generateElevationData()` function samples the route at 30-meter intervals:

```typescript
for (let i = 0; i < numSamples; i++) {
  const targetDistanceMeters = i * sampleIntervalMeters;
  const interpolatedPoint = interpolatePointAtDistance(
    directions[0].routes[0].geometry.coordinates,
    targetDistanceMeters,
  );

  if (interpolatedPoint) {
    const elevation = mapRef.queryTerrainElevation(interpolatedPoint);

    if (elevation !== null && elevation !== undefined) {  // <-- BUG: Skips point on null
      profileData.push({
        distance: targetDistanceMeters / 1000,
        elevation: elevation,
        coordinate: interpolatedPoint,
      });
    }
  }
}
```

**Bug Analysis**:
- If `queryTerrainElevation()` returns null for the first sample (distance=0), the graph won't start at 0
- If it returns null for middle samples, there will be gaps in the data
- The function silently drops data points instead of handling missing elevation gracefully

### 2. Point Interpolation (`components/map.tsx:198-236`)

The `interpolatePointAtDistance()` function finds coordinates at specific distances:

- Returns `null` if coordinates array has fewer than 2 points (line 202)
- Returns first coordinate for distance <= 0 (lines 205-207)
- Uses cumulative distance calculation with Haversine formula
- Edge case: If segment distance < 0.001, returns segment start point

**Potential issues**:
- If route geometry is malformed, may return null
- Distance calculation might have precision issues

### 3. Distance Calculation (`components/map.tsx:239-251`)

Standard Haversine formula implementation. This is well-tested and unlikely to be the bug source.

### 4. Data Flow Issues

The function only processes `directions[0].routes[0]`:
- `directions` is `Array<Directions>` but only first element is used
- `routes` array may have multiple routes, only first is used
- Multi-waypoint routes may not be handled correctly

## Potential Fixes (To Explore After Tests Confirm Bug)

1. **Interpolate missing elevations**: If `queryTerrainElevation()` returns null, interpolate from neighbors
2. **Fallback to 0**: Use a default elevation value instead of skipping
3. **Reduce sample interval**: Sample more frequently near problem areas
4. **Always include endpoints**: Ensure distance 0 and max distance are always in the result
